菜单栏详细介绍 - 8部分

  1. Charles

    • About Charles
      • 这里可以查看charles的版本
    • Preferences..
      • user interface 是用户界面的意思
        • Display font 是显示字体–用于在Charles中显示文本的字体。这个一般不用修改，除非用外国话和奇怪字符。
        • Display font size 是设置charles里面所有文字的字号大小。
        • show charlse icon in the menu bar 在菜单栏中显示charlse图标
        • charles window always on top charles窗口总是在最上面
        • Highlight changes in the structure view 高亮突出
        • show memory usage 显示内存使用情况。在Charles右下角会显示。
      • viewers
        • Combine header and body viewers 结合头部和返回体一起看
        • Combine request and response 结合请求和响应一起看
        • Speed 网速显示方式
        • Time 时间显示方式
      • Launch
        • start throttling 对应小乌龟，模拟快慢网速
      • Warnings
        • prompt to save session 保存绘会话提示：是否在关闭charles时提示用户保存会话。
        • warn when clearing a session 清除会话时发出警告
    • Services
      • Services Preferences.. 系统偏好设置
    • Hide Charles 桌面设置，隐藏charles
    • Hide Others 桌面设置，隐藏除了charles之外的打开的页面或者进程
    • Show All 隐藏之后点这个，就恢复展示了
    • Quit Charles 退出关闭charles进程

      • Sequence视图是根据时间来进行排序的，如上图所示，可以通过时间节点来找到返回数据。如果网址太多可以通过Filter 功能，可以输入关键字来快速筛选出 URL 中带指定关键字的网络请求。

      • URL 请求的url

      • Response Code 响应状态码

      • Notes 备注 方便记录一些接口的信息，在overview视图上可以看到备注信息
      • Timing

        • Request End Time 发送的最后一个请求的最后一个字节的时间
        • Response Start Time 响应开始的时间
        • Response End Time 响应结束的时间
        • Duration 整个请求响应的时间
        • DNS 所有选择的session解析DNS所花费的时间总和
        • Request 请求耗费时间
        • Response 响应耗费时间
        • Request Speed 请求的速度
        • Response Speed 响应的速度
      • Size
        • Request 请求的字节
          • Header 请求头的字节数
          • Query String 请求部分
        • Response 响应的字节书
          • Header 响应头的字节数
          • Body 响应部分
            • Contents 请求的内容
            • 上半部分为请求头、查询字符串、原始数据、Cookies、Raw
            • 下半部分为响应体
            • Summary 总结
            • resource 资源
            • host 主机
            • code 密码
            • mime type MIME类型
            • header 表头
            • body 征文
            • time 时间
            • total 全部
            • grand total 合计
            • duration 持续时间
            • Chart 表
            • resource 资源
            • timeline 时间轴
            • total duration 总持续时间
            • timeline 时间轴
            • size 大小
            • duration 持续时间
            • types 类型
            • flow 流量
    • Request 切换到content模块 com+2切换
    • Response 切换到content模块 com+3切换
    • Summary 就是主界面右侧顶部的三个tab之一 概要 com+2切换 com+4切换
    • Chart 就是主界面右侧顶部的三个tab之一 图表 com+3切换 com+5切换

  5. Proxy

    • Stop Recording(Session 1) 停止记录会话
    • Stop SSL Proxying 停止SSL代理
    • Start Throttling 开始节流
    • Enable Breakpoints 启用断点
    • Recording Settings.. 记录会话设置配置Charles的录制行为
      • Options
        • Recording Size Limits 当Charles记录时，请求和响应的头和主体存储在内存中，或写入磁盘上的临时文件。有时，内存中的数据量可能过多，Charles会通知您并停止记录。在这种情况下，应该清除Charles会话以释放内存，然后再次开始录制。在“记录设置”中，可以限制Charles记录的最大大小。这完全不会影响的浏览器，Charles只会停止记录。
      • Include 仅记录与以下位置之一匹配的请求。 如果此列表为空，则所有请求都将被记录，除非另行排除。
      • Exclude 与以下位置之一匹配的请求将不会被记录
    • SSL Proxying Settings.. SSL代理设置
      • SSL Proxying Charles 可以向您显示ssl请求和响应的纯文本内容,只有符合以下位置的站点才会被代理。
      • Server certificates 您必须创建一个charles安全存储才能将私有ssl证书导入charlse.
      • client certificates 客户端证书
      • root certificate 根证书
    • Throttle Settings.. 节流设置 （开启后，Charles界面的”绿色小乌龟”会在点亮状态，则表示已经开启限速了。网络上下行规则：http://www.doc88.com/p-3015598697603.html）
      • Throttle preset 预设置 提供了2/3/4G等设置参数值，也可以把预设置自定义参数保存下来，方面下次使用
      • bandeidth 带宽(千位/秒) 由该值控制数据参数速度
        • download—>下载速度，指的是从网络上下载下来的速度
        • upload—>上传速度，指的是把数据上传到网络上的速度
      • utilisation 可用率(%) 带宽可用率，大部分modern是100%。
      • round-trip latency 往返延迟(Ms)
        • 延迟，就是网络请求发起时，延迟多长时间去请求，例如设置10m，则发起请求时，延迟10秒，服务端才会接受到信息
      • Mtu 最大传输单位(字节)
        • 即TCP包的最大size，可以更真实模拟TCP层，每次传输的分包情况
      • Reliabilith 可靠性(%)
        • 指连接的可靠性。这里指的是10kb的可靠率。用于模拟网络不稳定。指连接的可靠性。这里指的是10kb的可靠率。用于模拟网络不稳定。
      • Stabilith 稳定性(%)
        • 连接稳定性，也会影响带宽可用性。用于模拟移动网络，移动网络连接一般不可靠。
      • Unstable quality range 不稳定质量范围(%)
      • Add Preset 添加限速预设（自定义2、3、4g）
      • Remove Preset 删除限速预设
    • Breakpoint Settings.. 断点设置 在发送和接收请求和响应之前对其进行拦截和编辑
    • Reverse Proxies.. 反向代理设置 将本地端口的HTTP和https请求代理到远程Web服务器。
    • Port Forwarding.. 端口转发 将本地tcp和udp端口转发到远程服务器。
    • macOS Proxy 记录计算机上的所有请求
    • Proxy Settings.. 代理设置
      • 代理服务器在两个端口上运行，一个用于HTTP代理，一个用于SOCKS代理。这些默认值通常很好。如果已经使用系统上的端口之一进行操作，则可能需要更改它们。
      • 动态代理端口使您可以让Charles在启动时选择可用的端口。如果您在多用户环境中运行Charles，这将很有用。
      • 传统上，Charles主要充当HTTP代理，同时还提供SOCKS代理功能。但是，使用HTTP代理时会出现一些性能问题，导致在使用或不使用Charles的情况下浏览器的行为都不同。在Charles中，SOCKS代理还不是默认代理，因为它只是刚刚可用。
      • Options bypass charles 这是使用代理（例如Charles）时要绕过的主机名的列表。该列表将传递到浏览器/ OS中的代理设置。通常可以使用一个空列表，但是如果有无法通过Charles访问的主机名，则可以在此处添加它们。“绕过域”列表是用空格分隔的主机和域列表。如果输入xk72.com之类的域，则大多数浏览器都将其表示为绕过* .xk72.com。
    • Access Control Settings.. 访问控制设置
      • 访问控制列表确定谁可以使用此Charles实例。通常是在自己的计算机上运行Charles的，并且只打算自己使用它，因此本地主机始终包含在ACL中。但是可能希望允许网络上的其他主机将Charles用作其代理。也许正在从多台计算机进行测试，并想在一个Charles实例中查看结果。可以输入单个IP地址或带子网掩码的IP地址。可以输入子网掩码作为IP地址，后跟掩码位数。192.168.0.0/16或作为IP地址，后跟子网掩码，例如 192.168.0.0/255.255.0.0。不在ACL中的主机第一次尝试访问Charles时，Charles中将显示一条警告消息（作为弹出消息）。通知访问尝试已被阻止，显示IP地址，并在要允许访问时提醒您将IP地址添加到ACL。每个IP地址只会显示一次警告。
    • External Proxy Settings.. 外部代理设置
      • 可能需要在网络上使用代理服务器才能访问Internet。在这种情况下，需要将Charles配置为在尝试访问Internet时使用现有代理。可以为以下内容配置单独的代理地址和端口：Http、HTTPS、socks。如果有SOCKS代理，Charles会将其用于所有非HTTP（S）通信，例如端口转发。
    • Web Interface Settings.. Web界面设置

  6. Tools

    • No Caching.. 禁用缓存设置 通过修改请求和响应头来防止缓存
    • Block Cookies.. 禁用Cookie设置 阻止cookie的发送和接收
    • Map Remote.. 远程映射设置 修改请求位置以将一个远程位置映射到另一个。
    • Map Local.. 本地映射设置
    • Rewrite.. 重写设置
      • 重写工具使您可以创建规则，以在请求和响应通过Charles时对其进行修改。诸如添加或更改标题或搜索和替换响应正文中的某些文本之类的规则。
    • Block List.. 黑名单设置
      • 阻止列表工具使可以输入应阻止的域名。当的网络浏览器尝试从列出的阻止域名中请求任何页面时，该页面将被阻止。也可以输入通配符以阻止子域。
    • Allow List.. 白名单设置
    • DNS Spoofing.. DNS欺骗
      • 有时想要测试域名，就好像它指向另一个IP地址一样。例如，如果设置了虚拟主机，但是IP地址更改尚未通过DNS传播，则可以对其进行欺骗并立即测试虚拟主机。Charles包含配置的域名到IP地址映射的列表。当请求提供列出的域名时，Spoof DNS插件会发现欺骗IP将请求重定向到该地址。主机HTTP标头保持不变，因此就好像DNS服务器返回了欺骗IP一样。
    • Mirror.. 镜像设置
    • Auto Save.. 自动保存
    • Client Process.. 客户端进程设置
    • Compose 编辑修改
      • 通过允许您在重复请求之前更改请求的内容，“编辑”工具以“重复”工具为基础。编辑时，可以编辑请求的不同部分，包括URL，标题，查询字符串或表单内容。
    • Compose New.. 撰写新的
    • Repeat 重复发包
      • 选择一个请求并重复一次。Charles将请求重新发送到服务器，并将响应显示为新请求。
    • Repeat Advanced.. 高级重复发包
      • “高级重复”工具扩展了“重复”工具，其中包含有关迭代次数和并发性的选项。这对于负载测试最有用。
    • Validate 验证
    • Publish Gist 发布要点
    • Import/Export Settings.. 导入/导出设置
    • Profiles.. 配置文件
    • Publish Gist Settings.. 发布要点设置 基本不用，发布代码到公网

  9. 页面工具按钮 从左往右的顺序介绍

    • 扫把：点击之后可清除抓取到的所有请求
    • 红色圆点：红色亮起的时候说明正在抓取，灰色的时候说明不在抓取状态
    • 小锁子：开始代理和停止代理
    • 乌龟：灰色乌龟是网速设置正常，绿色乌龟指网速限制开启了
    • 六角形：断电图标，灰色说明断电未开启，红色说明正在使用断点
    • 钢笔：编辑请求，点击后可以修改请求内容
    • 刷新：重复发送请求的图标，先选定某一请求，点击该图标，请求会再次被发送
    • 对号：Charles错误，所选内容不含任何可验证的响应
    • 扳手：对应于导航栏重的tools图标
    • 设置：从菜单中拿出来的常用charles设置
